# bgpd_nexthop_mgmt — model-based testing demo

Example for model-based generation of concrete test cases.

Tool Dependency: CrossHair (https://pypi.org/project/crosshair-tool/), for test generation.


## Nexthop Management Behavior

bgpd maintains two main states:
- rib: a mapping from prefix to resolved nexthop addresses. They stay constant in bgpd.
  However, the exact route issued from bgpd to the underlying hardware/kernel is dynamic:
  it depends on which nexthops are currently valid.
- nexthop_cache_table: the actual resolved status for each nexthop address. They are updated
  via calls to `api_bgp_nh_update`. After the update, bgpd re-issues route configurations
  according to the new validity status.


## Pipeline

```
         api_bgp_nh_update.py             the model — state + api behavior + invariant
                  |
                  v
         crosshair_target.py              wrapper for test generation
                  |
       [ gen_tests.sh : invoke crosshair ]
                  |
                  v
   tests/nh_update_argdict.txt            ~1000 raw inputs generated by CrossHair (path coverage)
                  |
       [ format_tests.py : dedupe + evaluate expected output & final state ]
                  |
                  v
   tests/nh_update_formatted.json         ~500 concrete test cases:
                                            
                  |
       [ gtest_stub : loads formatted .json ]
                  |
                  v
                ready for testing the actual C implementation (not shown in this demo)
```

## Layout

```
bgpd_nexthop_mgmt/
├── api_bgp_nh_update.py    # The model
├── crosshair_target.py     # Wrapper for symbolic execution
├── gen_tests.sh            
├── format_tests.py         # Postprocessor that generates concrete test cases
├── tests/
│   ├── nh_update_argdict.txt      # raw CrossHair output
│   ├── nh_update_formatted.json   # formatted cases
│   └── nh_update_pytest.py        # CrossHair output, represented in pytest files
└── gtest_stub/                    # Self-contained C++ reader for the JSON above
    ├── common_data.h              # Mirrors the Python dataclasses
    ├── json_reader.{h,cpp}        
    ├── main.cpp                   # Prints sample test cases
    └── Makefile                  
```

## Run

From `examples/`:

```
# 1. Generate raw inputs via CrossHair.
bgpd_nexthop_mgmt/gen_tests.sh

# 2. Format into concrete test cases.
python3 bgpd_nexthop_mgmt/format_tests.py
#   raw:      1049 inputs
#   deduped:  468 test cases
#   output:   bgpd_nexthop_mgmt/tests/nh_update_formatted.json

# 3. Build the C++ stub and load the JSON.
cd bgpd_nexthop_mgmt/gtest_stub
make
./read_tests
```

`format_tests.py` uses a `sys.path` bootstrap, so it runs from any cwd
(no `PYTHONPATH=.:lib` needed). `gen_tests.sh` still expects to be run
from `examples/`.


## Output schema

One JSON record per test case:

```json
{
  "TestId": 3,
  "Op": "api_bgp_nh_update",
  "InitialState": {
    "nexthop_cache_table": [
      {"endpoint": "<ipv6>", "resolved": "<ipv6>", "valid": false}
    ],
    "rib": [
      {"prefix": "<ipv6/N>", "nexthops": ["<ipv6>", "..."]}
    ]
  },
  "ApiParam": {"endpoint": "<ipv6>", "resolved": "<ipv6>", "status_up": true},
  "FinalState": { "...": "same shape as InitialState" },
  "DownstreamMsgs": [
    {"op": "SET", "prefix": "<ipv6/N>", "nexthops": ["<ipv6>"]}
  ]
}
```

## Dependencies

- Python ≥3.12 with `crosshair-tool` (`pip install crosshair-tool`)
- json-c for the C++ stub (`brew install json-c` on macOS)
